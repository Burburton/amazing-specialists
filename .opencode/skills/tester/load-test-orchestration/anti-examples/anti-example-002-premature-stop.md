# Anti-Example: Premature Stop

## 错误做法

过早停止负载测试，未达到稳态。

## 问题示例

```yaml
# 错误: 测试时间太短
load_test:
  concurrency: 1000
  duration: 30s  # 太短!
  
result:
  error_rate: 0%
  conclusion: "测试通过"
```

## 为什么这是错的

1. **未达稳态**: 系统可能还在预热
2. **隐藏问题**: 内存泄漏等问题需要时间暴露
3. **数据不足**: 样本量不足以得出结论
4. **遗漏场景**: 未覆盖完整业务周期

## 常见需要更长测试时间的场景

| 场景 | 建议时长 | 原因 |
|------|----------|------|
| 正常负载 | 5-10 分钟 | 达到稳态 |
| 峰值负载 | 10-15 分钟 | 观察稳定性 |
| 耐久测试 | 1-2 小时 | 发现内存泄漏 |
| 压力测试 | 15-30 分钟 | 找到极限 |

## 正确做法

```yaml
# 正确: 足够的测试时长
load_test:
  stages:
    - { duration: '1m', target: 100 }   # ramp-up
    - { duration: '5m', target: 100 }   # warm-up
    - { duration: '1m', target: 500 }   # ramp-up
    - { duration: '10m', target: 500 }  # steady state
    - { duration: '1m', target: 0 }     # ramp-down
    
  total_duration: 18m
    
  # 稳态判定
  steady_state_check:
    window: 5m
    max_variance: 10%
```

## 稳态判断标准

1. **响应时间稳定**: 最近 5 分钟波动 < 10%
2. **错误率稳定**: 最近 5 分钟波动 < 5%
3. **资源使用稳定**: 最近 5 分钟波动 < 5%
4. **无明显趋势**: 无持续上升或下降

## 测试时长建议

| 测试类型 | 最小时长 | 推荐时长 |
|----------|----------|----------|
| 冒烟测试 | 2 分钟 | 5 分钟 |
| 负载测试 | 5 分钟 | 10 分钟 |
| 压力测试 | 10 分钟 | 20 分钟 |
| 耐久测试 | 30 分钟 | 2 小时 |
